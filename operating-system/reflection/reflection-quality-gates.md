# Reflection Quality Gates

## Evidence Gate

Reflection deve se basear em arquivos, ADRs, validation reports, failed gates, unresolved risks, commits, changelog ou feedback.

## Specificity Gate

Findings devem ser específicos o suficiente para ação.

## Actionability Gate

Recomendações devem definir action, owner, target artifact, expected outcome e priority.

## Learning Gate

Toda sprint deve extrair ao menos uma lição sistêmica.
