# Non-Technical Builder Interview Playbook

## Goal

Help a non-technical user describe a real workflow in plain language.

## Steps

1. Ask about the problem story.
2. Ask about current workaround.
3. Identify people involved.
4. Identify information to store or search.
5. Map workflow steps and statuses.
6. Define first useful version.
7. Translate to software concepts.
8. Recommend no-code, low-code, AI-assisted app or professional project.

## Rule

Do not ask about stack, database or deployment before understanding the problem.
