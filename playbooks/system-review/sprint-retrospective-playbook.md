# Sprint Retrospective Playbook

## Objetivo

Revisar sprint concluída e criar aprendizado acionável.

## Inputs

Sprint plan, handoff, validation report, changelog, ADRs, files created/updated, unresolved risks, handoff results e feedback.

## Procedure

1. Summarize completed scope.
2. Compare planned versus actual.
3. Identify created artifacts.
4. Validate quality gates.
5. Identify decisions and risks.
6. Identify documentation drift.
7. Identify protocol gaps.
8. Create improvement backlog items.
9. Recommend next sprint adjustments.
