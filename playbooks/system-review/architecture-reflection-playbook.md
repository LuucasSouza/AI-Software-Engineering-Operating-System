# Architecture Reflection Playbook

## Run When

- Architecture artifact is created.
- ADR changes architecture.
- Implementation deviates from architecture.
- Scalability or security assumptions change.

## Questions

- Is architecture aligned with product scope?
- Are module boundaries clear?
- Are decisions documented?
- Are trade-offs still valid?
- Do ADRs need superseding?
- Are diagrams and integration contracts current?
