# Mode Selection Matrix

| Mode | User Profile | Primary Need | Output Style | Downstream Bias |
| --- | --- | --- | --- | --- |
| Non-Technical Builder | Does not know software terms | Guided problem interview | Plain language + translated artifacts | Discovery Intake Package |
| Vibe Coder | Uses AI tools but lacks engineering process | Guided build plan | Phases, prompts, guardrails | Build Brief + Discovery Intake |
| Professional Engineer | Understands engineering | Technical planning | Full engineering artifacts | Technical Intake |

## Selection Rule

Prefer explicit selection. Use inferred selection only when the user does not choose a mode.
