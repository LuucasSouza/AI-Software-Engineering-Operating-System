# Frameworks Consolidation Strategy

## Objetivo

Transformar engines operacionais em um sistema de frameworks ensinável, governável, reutilizável e extensível.

## Princípios

- No duplicate frameworks.
- Explicit interface contracts.
- Frameworks must be engine-aware.
- Frameworks must be agent-executable.
- Frameworks must be human-reviewable.
- Frameworks must scale by depth.
- Frameworks must support evolution.

## Distinções

- Engine: capacidade operacional.
- Framework: sistema conceitual reutilizável.
- Protocol: procedimento executável.
- Template: estrutura de artefato.
- Playbook: guia aplicado.
- Agent: ator com responsabilidades.

## Próximos passos

- Validar frameworks com o Quality Assurance Framework.
- Aplicar o meta-framework no reference project.
