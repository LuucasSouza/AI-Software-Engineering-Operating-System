# Discovery-to-Delivery Framework

## Objetivo

Conduzir um projeto de software do intake até um plano executável, documentado e continuamente melhorável.

## Stage Outputs

| Stage | Outputs |
| --- | --- |
| Intake | Intake brief, assumptions, unknowns |
| Discovery | Discovery document, constraints, validation gaps |
| Product | PRD, MVP scope, roadmap |
| Architecture | Architecture overview, domain model, integration map |
| Decision | Decision matrix, ADRs, confidence |
| Risk | Risk register, mitigations |
| Optimization | Optimization review |
| Execution | Execution plan, work packages |
| Documentation | Documentation index, changelog |
| Handoff | Handoff package, receipt |
| Reflection | Retrospective, improvement backlog |
