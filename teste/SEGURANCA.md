# Seguranca no teste

Este teste nao deve usar dados sensiveis.

## Nao use ou copie

- `.env` real
- tokens
- senhas
- chaves de API
- backups
- banco de dados
- dados de cliente
- dados pessoais
- codigo privado que nao possa ser compartilhado

## Como registrar problemas

Se aparecer algo sensivel, nao copie o conteudo no feedback.

Escreva apenas o tipo do problema, por exemplo:

```text
Encontrei um arquivo .env real no projeto. Nao copiei o conteudo.
```

## Envio do feedback

O Resolve Aí nao deve enviar nada automaticamente.

O arquivo `feedback.md` sera enviado manualmente pelo participante para o Lucas.
