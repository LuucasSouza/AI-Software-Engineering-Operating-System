# Roadmap Update — Phase 2.5 and Phase 3

## 1. Purpose

This document defines how the roadmap must be updated after choosing the public name Resolve Aí.

## 2. Current state

The project completed:

- Sprint 0 — Foundation;
- Sprint 1 — Core Identity and Discovery;
- Sprint 2 — Product and Architecture;
- Sprint 3 — Decision, Risk and Optimization;
- Sprint 4 — Execution, Documentation, Handoff and Reflection;
- Sprint 5 — Framework Consolidation;
- Sprint 5.5 — Entry Modes Layer;
- Sprint 6 — Complete Template Packs;
- Sprint 7 — Final Consolidation;
- Phase 2 — Validation and Productization.

## 3. Insert Phase 2.5

Add:

```text
Phase 2.5 — Naming, Brand and Product Positioning
```

Purpose:

```text
Rename and reposition the project from AI-SEOS to Resolve Aí before creating the runtime layer.
```

## 4. Move Runtime Productization after Phase 2.5

Phase 3 becomes:

```text
Phase 3 — Resolve Aí Runtime Productization
```

Purpose:

```text
Transform Resolve Aí from framework documentation into a tool that can be activated inside any project through CLI, agent instructions, adapters, and future MCP/runtime capabilities.
```

## 5. Phase 2.5 deliverables

- Brand identity;
- README update;
- rename strategy;
- migration guide;
- public positioning;
- future CLI naming;
- communication guide;
- ADRs;
- Phase 3 handoff.

## 6. Phase 3 updated direction

Phase 3 should use the new product identity:

```text
Resolve Aí
```

Future CLI:

```text
resolve-ai
```

Future local docs path:

```text
docs/resolve-ai/
```

Legacy compatibility:

```text
docs/ai-seos/
```

## 8. Roadmap note

The roadmap should explicitly say:

> Phase 2.5 was inserted before Phase 3 to avoid building runtime commands, installation flows, and public onboarding under a name that would immediately need to be changed.

## 9. Definition of Done

Roadmap update is complete when:

- Phase 2.5 appears between Phase 2 and Phase 3;
- Phase 3 is renamed to Resolve Aí Runtime Productization;
- future part numbering is updated;
- README and CHANGELOG mention the rename;
- ADR records the roadmap adjustment.
