# Phase 11 ADRs

Create ADRs 0166 to 0175.

## ADR 0166 — Adopt External Alpha Validation Before npm Publication

Decision: Require external alpha validation before npm publication.

## ADR 0167 — Adopt Three-Profile Validation Model

Decision: Validate Non-Technical Builder, Vibe Coder and Professional Engineer profiles.

## ADR 0168 — Adopt Real Project Validation Matrix

Decision: Validate empty, Node, React/Vite, in-progress and legacy-like project types.

## ADR 0169 — Adopt Portuguese UX Validation as Release Gate

Decision: Command names and messages must be evaluated for Brazilian Portuguese clarity.

## ADR 0170 — Adopt Alpha Feedback Triage

Decision: Use P0/P1/P2/P3 feedback severity.

## ADR 0171 — Adopt No Sensitive Content Capture Rule

Decision: Validation may report paths/names but must not capture sensitive contents.

## ADR 0172 — Adopt Alpha Readiness Scorecard

Decision: Use 100-point alpha readiness scorecard with blocker override.

## ADR 0173 — Adopt GitHub Alpha Issue Templates

Decision: Create issue templates for alpha feedback, CLI bugs, UX confusion, docs improvement and safety concerns.

## ADR 0174 — Adopt Private/Controlled Alpha Before Public Distribution

Decision: Controlled alpha validation precedes broader public distribution.

## ADR 0175 — Adopt Release Recommendation Record

Decision: Every alpha validation round must end with proceed/hold/extend recommendation.
