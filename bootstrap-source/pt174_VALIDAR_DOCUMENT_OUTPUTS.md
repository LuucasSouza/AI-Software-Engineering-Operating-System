# pt174 — `validar` Document Outputs

## Objective
Define the canonical documents generated by `resolve-ai validar`.

## Output directory

```text
docs/resolve-ai/
```

## Overwrite policy
Do not overwrite existing files by default.

If a file already exists, create a timestamped variant or print a clear message depending on the current CLI convention.

## 27-checklist-pos-execucao.md
Purpose: create a practical checklist for the user or agent.

Required sections:

```markdown
# Checklist Pós-Execução

## Checklist técnico
- [ ] O projeto compila?
- [ ] Os testes relevantes foram executados?
- [ ] A funcionalidade principal foi verificada manualmente?
- [ ] Nenhum arquivo sensível foi adicionado?
- [ ] A documentação necessária foi atualizada?

## Checklist de produto
- [ ] O resultado resolve a tarefa preparada?
- [ ] O comportamento esperado está claro?
- [ ] Existe alguma regressão aparente?

## Checklist de segurança
- [ ] Nenhum segredo foi exposto?
- [ ] Permissões continuam corretas?
- [ ] Dados pessoais não foram adicionados indevidamente?
```
