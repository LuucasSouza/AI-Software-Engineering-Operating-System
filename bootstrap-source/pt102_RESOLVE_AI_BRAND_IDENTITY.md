# Resolve Aí — Brand Identity

## 1. Name

The public name of the project is:

# Resolve Aí

The name is intentionally informal, direct, Brazilian, memorable, and action-oriented.

It sounds like a command and a promise at the same time.

## 2. Core promise

> Me dá o problema ou a ideia, e eu te ajudo a resolver.

This sentence is the emotional and product center of the project.

It communicates that the user does not need to start with technical vocabulary. They can start with a problem, a messy project, a vague idea, or a daily operational pain.

Resolve Aí translates that into a structured path.

## 3. Tagline options

Primary tagline:

> Me dá o problema ou a ideia, e eu te ajudo a resolver.

Short tagline:

> Da ideia ao sistema, sem bagunça.

Developer-oriented tagline:

> Engenharia de software para construir com IA sem perder o controle.

Non-technical tagline:

> Você explica o problema. A IA ajuda a organizar e construir.

Runtime tagline:

> Instale no projeto. Rode o diagnóstico. Resolva com método.

## 4. Brand essence

Resolve Aí should feel:

- practical;
- helpful;
- clear;
- slightly informal;
- trustworthy;
- technically serious underneath;
- accessible to non-technical people;
- useful to programmers;
- friendly without being childish;
- ambitious without sounding corporate.

## 5. What Resolve Aí is

Resolve Aí is a framework and future runtime layer that helps users turn problems, ideas, and existing software projects into structured, well-planned software solutions using AI.

It combines:

- guided interviews;
- problem discovery;
- product definition;
- architecture review;
- decision records;
- risk analysis;
- execution planning;
- documentation;
- handoff between AI agents;
- project diagnosis;
- future CLI/runtime automation.

## 6. What Resolve Aí is not

Resolve Aí is not just a prompt pack.

Resolve Aí is not just a README.

Resolve Aí is not a random vibe coding script.

Resolve Aí is not a replacement for human judgment.

Resolve Aí is not only for professional engineers.

Resolve Aí is not a clone of Context Mode.

It may be inspired by the ease of use of tools such as Context Mode, but its mission is different: software engineering orchestration, not only context optimization.

## 7. Public description

Short version:

> Resolve Aí helps you turn an idea, a problem, or an existing project into a clear software plan using AI.

Medium version:

> Resolve Aí is an open-source framework for people who want to build software with AI without starting from chaos. It interviews the user, understands the problem, diagnoses existing projects, proposes solutions, organizes architecture, maps risks, creates plans, and prepares handoffs for AI coding agents or human developers.

Technical version:

> Resolve Aí is a software engineering operating framework for AI-assisted development, composed of entry modes, engines, protocols, templates, ADRs, readiness scorecards, handoff contracts, and future runtime tooling.

## 8. Audience language

### Non-Technical Builder

Use simple language.

Avoid unexplained words like repository, backend, deployment, CI/CD, ORM, provider, serverless, multi-tenant, and API.

Explain with examples.

### Vibe Coder

Use practical language.

Focus on what to do next, what to avoid, what to ask the AI agent, and how to keep the project from becoming messy.

### Professional Engineer

Use precise technical language.

Include architecture, trade-offs, ADRs, constraints, risks, handoff contracts, readiness levels, and validation gates.

## 9. Naming rule

Use:

```text
Resolve Aí
```

for public brand and human-facing docs.

Use:

```text
resolve-ai
```

for future CLI/package/technical command references.

Use:

```text
AI-SEOS
```

only when referring to the historical origin or internal architecture lineage during migration.
