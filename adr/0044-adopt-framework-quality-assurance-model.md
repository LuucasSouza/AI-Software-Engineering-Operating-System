# ADR-0044 — Adopt Framework Quality Assurance Model

## Status

Accepted

## Context

Framework quality must be measured by usefulness, executability and traceability, not document length.

## Decision

Adopt a framework QA model with dimensions, scoring, critical gates and validation report template.

## Alternatives Considered

- Rely on manual proofreading.
- Defer quality model until automation exists.

## Consequences

Frameworks can be reviewed consistently before promotion.

## Risks

Manual scoring may be inconsistent until examples and automation exist.

## Reversal Strategy

Use the QA model as a checklist until scoring is validated by reference implementations.
