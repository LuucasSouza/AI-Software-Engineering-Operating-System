# ADR-0043 — Adopt Framework Governance and Versioning

## Status

Accepted

## Context

Frameworks must evolve without losing coherence or trust.

## Decision

Adopt framework lifecycle statuses, metadata, semantic versioning, proposal process, review board and deprecation policy.

## Alternatives Considered

- Change frameworks directly without governance.
- Use only repository-level governance.

## Consequences

Framework evolution becomes traceable and maintainable.

## Risks

Governance can become too heavy for small contributions.

## Reversal Strategy

Keep editorial changes lightweight and require ADRs only for structural changes.
