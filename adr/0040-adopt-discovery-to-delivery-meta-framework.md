# ADR-0040 — Adopt Discovery-to-Delivery Meta-Framework

## Status

Accepted

## Context

Users need one end-to-end explanation for applying AI-SEOS from idea to delivery readiness.

## Decision

Adopt the Discovery-to-Delivery Meta-Framework with stages 0 to 10 and lightweight, standard and enterprise operating paths.

## Alternatives Considered

- Keep each engine self-contained without one project path.
- Create only examples without a meta-framework.

## Consequences

Onboarding and project application become clearer.

## Risks

Users may treat the path as waterfall unless feedback loops are emphasized.

## Reversal Strategy

Split the meta-framework into smaller path-specific guides if one document becomes too broad.
