# ADR-0042 — Adopt Project Readiness Scorecards

## Status

Accepted

## Context

Stage transitions require objective evidence that a project is ready to proceed.

## Decision

Adopt project and engine readiness scorecards across the full lifecycle.

## Alternatives Considered

- Use informal judgment.
- Use only binary quality gates.

## Consequences

Readiness becomes visible, comparable and actionable.

## Risks

Scorecards may be treated as false precision.

## Reversal Strategy

Use qualitative recommendations while keeping required artifacts and blocking conditions.
