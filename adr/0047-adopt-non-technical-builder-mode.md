# ADR-0047 — Adopt Non-Technical Builder Mode

## Status

Accepted

## Context

AI-SEOS must serve people who want to solve real problems but do not know software terminology.

## Decision

Adopt Non-Technical Builder Mode as an official entry mode using plain-language interview and internal software translation.

## Alternatives Considered

- Require all users to learn technical vocabulary.
- Provide only professional engineering templates.

## Consequences

AI-SEOS becomes accessible to non-technical users while preserving downstream engineering rigor.

## Risks

The mode may oversimplify if translation quality gates are weak.

## Reversal Strategy

Keep non-technical intake as a playbook if it should not remain a first-class mode.
