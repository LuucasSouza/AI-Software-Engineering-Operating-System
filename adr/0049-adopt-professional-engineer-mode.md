# ADR-0049 — Adopt Professional Engineer Mode

## Status

Accepted

## Context

Professional engineers need precise artifacts, constraints, alternatives, ADRs and risk analysis.

## Decision

Adopt Professional Engineer Mode as the rigorous technical entry path.

## Alternatives Considered

- Use one generic mode for all technical users.
- Make all templates non-technical first.

## Consequences

AI-SEOS preserves depth for engineers while adding accessibility for other users.

## Risks

Professional mode may become overly verbose if not guided by quality gates.

## Reversal Strategy

Refine professional templates by project depth rather than removing the mode.
