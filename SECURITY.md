# Security Policy

## Objetivo

Definir como vulnerabilidades, riscos de segurança e problemas de uso indevido devem ser reportados e tratados no AI-SEOS.

## Escopo

Esta política cobre documentos, protocolos, templates, agentes, exemplos, automações futuras e qualquer material publicado neste repositório.

## Não Escopo

Esta política não substitui auditoria formal, revisão jurídica, compliance regulatório ou threat modeling específico de projetos que usem o AI-SEOS.

## Princípios

- Segurança deve ser considerada desde discovery e arquitetura.
- Issues sensíveis não devem ser abertas publicamente antes de triagem.
- Agentes de IA não devem expor segredos, dados pessoais ou material confidencial.
- Mudanças de segurança relevantes devem ser registradas no changelog.

## Como reportar

Enquanto o projeto não tiver canal dedicado, reporte problemas de segurança aos maintainers por canal privado definido pelo dono do projeto.

Inclua:

- descrição do problema;
- impacto esperado;
- passos de reprodução, se aplicável;
- arquivos afetados;
- mitigação sugerida;
- urgência percebida.

## Responsabilidades

Core Maintainers devem triar, priorizar, corrigir e documentar riscos de segurança. Contributors devem evitar publicar segredos, dados sensíveis ou exemplos inseguros.

- Criar template de issue de segurança quando houver GitHub público.
- Criar protocolo de threat modeling nas próximas sprints.
