# Documentation Maintenance Protocol

## Objetivo

Manter documentação alinhada a engines, protocolos, templates, ADRs, changelog e roadmap.

## Triggers

- ADR criada ou superseded.
- Engine, protocolo ou template criado ou alterado.
- Sprint scope alterado.
- Architecture, risk ou release scope alterados.
- Handoff ou onboarding alterado.

## Fluxo

1. Listar arquivos criados e atualizados.
2. Validar índices.
3. Validar metadata.
4. Validar ADR index.
5. Validar changelog.
6. Validar roadmap.
7. Identificar drift.
8. Criar remediation tasks.
