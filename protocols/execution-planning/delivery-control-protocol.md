# Delivery Control Protocol

## Objetivo

Controlar execução em cada iteração e detectar drift.

## Checklist

- [ ] Completed work registrado.
- [ ] Blocked work registrado.
- [ ] Scope changes classificados.
- [ ] Decision changes encaminhados para Decision Engine.
- [ ] New risks encaminhados para Risk Engine.
- [ ] Architectural deviations revisados.
- [ ] Documentation changes identificados.
- [ ] Test status registrado.
- [ ] Release readiness atualizado.

## Change Control

Mudanças em escopo, arquitetura, segurança, custo, prazo ou contratos públicos exigem atualização do artefato correspondente e possível ADR.
