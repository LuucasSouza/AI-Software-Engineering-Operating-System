# Information Architecture

## Objetivo

Organizar documentação por propósito e nível de abstração.

## Hierarquia

| Level | Purpose | Examples |
| --- | --- | --- |
| 0 | Root project documents | README, ROADMAP, GOVERNANCE |
| 1 | Domain indexes | operating-system/README.md |
| 2 | Module indexes | operating-system/execution/README.md |
| 3 | Canonical documents | execution-engine.md |
| 4 | Generated/project artifacts | sprint reports, examples |

## Princípios

- One concept, one canonical home.
- Cross-link instead of copying.
- Indexes must be curated.
- Documentation must age visibly.
