# Documentation Standards

## Objetivo

Definir regras comuns para documentos do AI-SEOS.

## Regras

- Use kebab-case para arquivos canônicos internos.
- Use uppercase apenas para documentos raiz tradicionais.
- Use um único H1 por documento.
- Documentos longos devem usar seções numeradas.
- README deve explicar objetivo, escopo, conteúdo, exclusões e relação com o sistema.
- Changelog deve registrar mudanças significativas, não cada operação de arquivo.
- Mermaid deve ser usado para lifecycle, flow, dependency maps e state transitions quando útil.

## Estrutura recomendada

Purpose, Context, Scope, Non-scope, Principles, Inputs, Outputs, Lifecycle, Object Model, Quality Gates, Roles, Examples, Anti-patterns, Definition of Done e Related artifacts.
