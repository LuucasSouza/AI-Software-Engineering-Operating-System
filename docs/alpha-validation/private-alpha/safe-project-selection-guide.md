# Safe Project Selection Guide

## Prefer

- Temporary directories.
- Simulated projects.
- Sanitized copies.
- Personal experiments without secrets.
- Example repositories created for validation.

## Avoid

- Production repositories.
- Client repositories.
- Repositories with secrets.
- Databases, dumps, backups, exports, or spreadsheets with real people.
- Projects under NDA unless explicitly sanitized.

## Per Profile

| Profile | Recommended Project |
|---|---|
| Non-Technical Builder | Empty directory plus narrated idea |
| Vibe Coder | Small Node/Vite fake app or sanitized AI-built project |
| Professional Engineer | Example technical repository or sanitized internal tool |

## Stop Conditions

Stop the session if credentials, private customer data, or confidential material appear.
