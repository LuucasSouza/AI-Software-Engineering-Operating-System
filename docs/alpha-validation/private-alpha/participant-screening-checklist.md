# Private Alpha Participant Screening Checklist

## Required Checks

- [ ] Participant matches one official profile.
- [ ] Participant understands this is private alpha.
- [ ] Participant agrees not to use sensitive data.
- [ ] Project is simulated, sanitized, or safe for test.
- [ ] Participant understands feedback will be anonymized.
- [ ] Participant understands `resolver` does not execute code automatically.
- [ ] Participant understands no npm publish, tag, release, deploy, commit, or push will happen.

## Exclusion Criteria

- [ ] Project contains real credentials or `.env` content.
- [ ] Project contains customer data.
- [ ] Participant expects full automated implementation.
- [ ] Participant needs production support.
- [ ] Participant cannot use a safe test scenario.
