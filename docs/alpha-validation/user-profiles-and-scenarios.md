# User Profiles and Scenarios

## Objetivo

Definir como a validacao alpha cobre os tres perfis publicos do Resolve Ai.

## Escopo

Os cenarios conectam perfil, tipo de projeto, comandos esperados e sinais de sucesso.

## Matriz

| Perfil | Projeto representativo | Foco da validacao | Sucesso esperado |
|---|---|---|---|
| Non-Technical Builder | Diretorio vazio ou ideia inicial | Clareza, linguagem simples e proximo passo | Usuario entende o que foi criado e nao precisa escolher stack |
| Vibe Coder | Vite/React ou app parcialmente criado | Reducao de caos, backlog e prompt para agente | Usuario recebe plano acionavel e limites de seguranca |
| Professional Engineer | Node/projeto maduro simulado | Rigor, rastreabilidade e risco | Usuario confia nos limites e na utilidade tecnica |
| Projeto em Andamento | Repositorio com arquivos existentes | Diagnostico e continuacao | Resolve Ai nao trata como greenfield puro |

## Principios

- A experiencia publica e em portugues.
- O mesmo runtime atende perfis diferentes por profundidade e recomendacao.
- A recomendacao de modo deve ser explicavel.

## Responsabilidades

Maintainers devem mapear cada feedback ao perfil correto antes de decidir prioridade.

## Proximos passos

Adicionar evidencias de usuarios reais para cada perfil.
