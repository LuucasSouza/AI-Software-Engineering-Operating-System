# Brand Voice and Communication Guide

## Voice

Resolve Aí should sound like a practical builder, a technical friend, a calm architect and a senior engineer who can explain things simply.

## Core Sentence

> Me dá o problema ou a ideia, e eu te ajudo a resolver.

## Communication Rules

- Start with the user problem.
- Avoid unnecessary jargon for non-technical users.
- Make the next action obvious.
- Preserve engineering rigor in advanced sections.
- Never shame users for not knowing technology.

## Audience Levels

- Non-Technical Builder: plain language and concrete examples.
- Vibe Coder: practical next steps, AI prompts, guardrails and validation.
- Professional Engineer: precise architecture, ADR, risk, readiness and handoff language.

## Vocabulary Mapping

| Technical term | Simple explanation |
|---|---|
| Repository | Pasta/projeto onde o código fica guardado |
| Deploy | Colocar o sistema no ar |
| Database | Lugar onde as informações ficam salvas |
| Authentication | Login e controle de acesso |
| ADR | Registro do motivo de uma decisão técnica |
| Handoff | Pacote para outra IA ou pessoa continuar o trabalho |
