# v0.1.0-alpha Phase 19 GO/NO-GO Decision

## Decision

**PUBLIC ALPHA PUBLISHED (92/100).** GO executado — exclusivamente como public alpha. **Não é stable.**

## O que foi executado

- `npm publish ./packages/resolve-ai-cli --tag alpha` → `resolve-ai@0.1.0-alpha.0` (https://www.npmjs.com/package/resolve-ai).
- Tag `v0.1.0-alpha` criada após sucesso do npm e enviada ao origin.
- GitHub Release criado como pre-release (`--prerelease --latest=false`): https://github.com/LuucasSouza/Resolve-Ai/releases/tag/v0.1.0-alpha
- Install tests aprovados: global, npx e mini-fluxo em pasta limpa.

## Por que GO (alpha) foi permitido

- Aprovação explícita do mantenedor registrada no gate (2026-07-08, "Quero apertar o botão").
- Todos os gates técnicos verdes: 87/87 testes, smoke, dry-run idêntico ao auditado, `git diff --check`, nome disponível, versão inédita.
- Gates de bloqueio funcionaram de verdade: a execução parou duas vezes (ENEEDAUTH e EOTP) e só continuou com ação do mantenedor.

## Por que NÃO é stable

- Validação humana real permanece limitada (Non-Technical Builder pendente; Professional Engineer dedicado insuficiente).
- ADR-0274: proibido declarar stable sem validação humana real; score travado em 92/100.
- Known limitations continuam públicas no README, release notes e GitHub Release (ADR-0270).

## Ressalvas registradas

- Dist-tag `latest` aponta para `0.1.0-alpha.0` por mecânica do npm no primeiro publish (não removível). Documentação instrui sempre `@alpha`. Será corrigido naturalmente na primeira versão futura publicada como estável.
- `repository.url` normalizado pelo npm no publish (cosmético; `npm pkg fix` em fase futura).

## Next Action
