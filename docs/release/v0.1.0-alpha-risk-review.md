# v0.1.0-alpha Risk Review

## Riscos corrigidos

- Stack detection basica agora cobre Node.js, TypeScript, Vite e React.
- Heuristicas simples adicionadas para Firebase, Supabase, Python e monorepo.
- Projeto novo sem testes/CI nao vira risco critico por esse motivo isolado.
- Arquivos sensiveis por nome/caminho continuam como risco alto/critico sem copia de conteudo.

## Riscos restantes

| Risco | Severidade | Mitigacao |
|---|---|---|
| Sem validacao real de usuario | P1 | Rodar real user validation antes de npm |
| Heuristica pode errar stack | P2 | Linguagem usa "stack provavel" |
| Build usa strip simples de TypeScript | P2 | Manter testes e avaliar toolchain TS futura |
| Pacote npm ainda nao publicado | P1 | Gate manual na Phase 13 |
| `npm pack --dry-run` inclui `src/` e `tests/` | P2 | Revisar campo `files` antes de npm publico |

LIMITED GO para alpha controlada, sem npm publish.
