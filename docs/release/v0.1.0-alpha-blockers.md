# v0.1.0-alpha Blockers

## Objetivo

Registrar blockers conhecidos antes de publicacao alpha.

## P0 Blockers

Nenhum P0 observado na validacao local simulada da Phase 11.

## P1 Antes de npm publico

- Validar com pelo menos um usuario real por perfil.
- Confirmar nome, metadata e estrategia do pacote npm.
- Revisar UX de instalacao em ambiente limpo.
- Atualizar release notes com achados de usuarios reais.

## P1 Corrigidos na Phase 12

- Deteccao/exibicao de stack para Node.js, TypeScript, Vite e React.
- Heuristicas simples para Firebase, Supabase, Python e monorepo.
- Copy do `resolver` mencionando `validar` como comando existente.
- Hardening agressivo em projeto novo simples.

## Principios

- P0 bloqueia release.
- P1 bloqueia publicacao npm ampla, mas nao bloqueia validacao privada.
- Riscos devem permanecer visiveis.

## Responsabilidades

Maintainers atualizam este arquivo antes de tag ou publicacao.

## Proximos passos

Converter P1 em issues rastreaveis para Phase 12.
