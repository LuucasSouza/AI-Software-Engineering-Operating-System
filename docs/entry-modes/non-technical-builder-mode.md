# Non-Technical Builder Mode

## Objetivo

Permitir que uma pessoa sem conhecimento de software transforme um problema real em uma solução estruturada.

## Usuário alvo

Pessoas que usam planilhas, WhatsApp, papel, memória ou processos manuais e querem resolver um problema com software, automação, no-code, low-code ou ajuda de IA.

## Princípio

Comece pela vida, não pela tecnologia.

## Estilo

Simples, respeitoso, prático, guiado e sem jargão.

## Interview Phases

- Problem story.
- Current workaround.
- People involved.
- Information.
- Workflow.
- First useful version.

## Saída

Builder Summary com explicação em linguagem simples e tradução técnica para AI-SEOS.

## Quality Gates

- Problema compreensível.
- Workaround atual conhecido.
- Usuários alvo identificados.
- First useful version definido.
- Software translation gerada.
- Próximo passo claro.
