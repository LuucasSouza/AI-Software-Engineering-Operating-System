# Phase 5 Handoff

## Objetivo

Implementar `resolve-ai diagnosticar` como primeiro comando útil do runtime.

## Entregue

- Comando `diagnosticar`.
- Aliases `diagnostico` e `diagnóstico`.
- Heurísticas locais de projeto, stack, modo, riscos e maturidade.
- Geração de `docs/resolve-ai/00` a `09`.
- Atualização do último diagnóstico em `.resolve-ai/state.json`.
- `status` com resumo do último diagnóstico.
- Testes automatizados para cenários obrigatórios.
- ADRs 0106 a 0115.

## Limites Mantidos

- Sem MCP.
- Sem hooks.
- Sem telemetria.
- Sem APIs externas.
- Sem alteração de código do produto.
- Sem migração automática de `docs/ai-seos/`.
